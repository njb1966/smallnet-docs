# SmallNet Docs — Contribution Policy

Contact: `smdocs@pm.me`

## Ways to contribute

### 1) GitHub Pull Requests (preferred)
- Fork the repo
- Create a branch
- Make changes
- Open a Pull Request with a short description

Why PRs:
- clear, reviewable diffs
- public audit trail
- easy discussion and iteration

### 2) GitHub Issues (suggestions & corrections)
If you don’t want to use Git/PRs:
- open an Issue with:
  - link
  - suggested description/correction
  - what page it belongs on

An editor will translate it into a repo change.

### 3) Email suggestions
Email `smdocs@pm.me` with:
- the link
- what you think should change/add
- any relevant context

(Email suggestions may be turned into GitHub Issues/PRs by an editor.)

## What makes a good submission
For a new listing, include:
- Title
- URL
- Audience (newcomer/operator/dev/mixed)
- 1–3 sentence summary (neutral)
- Date you checked it worked (for `Last reviewed`)

## Style
- Keep tone neutral and factual.
- Avoid “best/worst/everyone should”.
- Prefer describing what something is for and who it helps.

## Review process
- We aim to review PRs weekly.
- Editors may request changes for clarity, tone, or missing info.
- Small fixes (typos, broken links) are very welcome.
