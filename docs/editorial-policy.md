# SmallNet Docs — Editorial Policy

## 1. What this is
SmallNet Docs is a **best-effort**, Gemini-native documentation hub for the small web (Gemini-first). It is **not official**. It may contain mistakes. Corrections are welcome.

## 2. “Map, not a judge”
We aim to be a *map of what exists and how it works*, not a ranking system.

We avoid:
- “best”, “top”, “must-use”, “the only good option”

We allow neutral descriptors/tags like:
- `beginner-friendly`
- `advanced`
- `GUI` / `TUI`
- `actively maintained` / `archived`
- `operator-focused`

## 3. Inclusion criteria
A resource may be included if it is:
- relevant to Gemini or small web usage,
- describable in a neutral way,
- reasonably stable (or clearly labeled if experimental),
- verifiable (we can load it and confirm it broadly matches the summary).

## 4. Review and freshness
Pages and listings include:
- `Last reviewed: YYYY-MM-DD`

Meaning: an editor checked for obvious issues (broken links, misleading summaries, major drift).
It is not a guarantee of correctness.

## 5. Mirroring policy
Default behavior is **linking**, not mirroring.

Mirroring is allowed only when:
- the license clearly permits it, OR
- the author explicitly grants permission, AND
- we include attribution and a source link.

## 6. Handling disagreements
If a maintainer disagrees with a description:
- we will correct factual errors promptly,
- we will label opinion as opinion or remove it,
- if needed, we will remove the listing rather than argue.

## 7. Removal policy
We may remove or de-list resources that are:
- dead links (after reasonable attempts),
- clearly malicious or abusive,
- predominantly spam,
- legally problematic to host/mirror.

When possible, removals are noted in the changelog.
